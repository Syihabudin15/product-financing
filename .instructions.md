## Best Practices

1. **When suggesting code**: Always consider the existing component structure and styling patterns (Tailwind CSS)

2. **When using MCP Paper**:
   - Use for finding research on best practices
   - Reference papers when suggesting architecture patterns
   - Cite sources for technical recommendations

3. **Code Quality**:
   - Follow TypeScript strict mode
   - Use proper type definitions
   - Maintain responsive design patterns
   - Keep components modular and reusable

4. **Development Workflow**:
   - Run `npm run dev` to start development server
   - Check for TypeScript errors with `get_errors`
   - Use `npm run build` for production builds
